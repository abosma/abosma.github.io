Het gebruik van AWS voor Big Data

Waarom Amazon's web services het uitvoeren van Big Data Projecten veel makkelijker kan maken

Geschreven door Atilla Bosma op 25 Mei, 2019

Wat is een van de nadelen van Big Data?

Een van de nadelen van Big Data is dat er veel data is dat verwerkt moet worden, dit betekent dus dat:

  1. Je kunt geen relationele databases gebruiken.
  2. Je kunt deze data niet op 1 computer verwerken.
  3. Je moet goed uitkijken hoe je met jouw data omgaat.

Gelukkig bestaan er velen verschillende opties voor het oplossen van deze problemen, en voor het uitvinden van hoe een van deze opties met deze problemen kan omgaan heb ik de gastles gevolgd van Dick Eimers over het gebruik van AWS voor Java.

De gastles ging specifiek over het gebruik van AWS in combinatie met het programmeertaal Java, in deze blog zal ik het hebben over wat AWS ons biedt en niet over hoe AWS in combinatie met Java gebruikt wordt sinds deze tools niet afhankelijk zijn van de programmeertaal.

Amazon en AWS

Laten we het eerst hebben over AWS, AWS staat voor Amazon Web Services en is een grote groep van tools die Amazon aanbiedt voor gebruik.

De kosten voor hun tools zijn gevarieerd gebaseerd op hoeveel de tool gebruikt wordt en hoeveel data er langs de tool gaat, als je de tool niet gebruikt kost het ook geen geld.

Een lijst van bijna alle tools die zij hebben.

Het belangrijke aan deze tools is dat deze serverless zijn (servers hoef jij niet te hosten voor de tools), ze automatisch meer hardware aanbieden gebaseerd op de vraag (de tool zal altijd goed werken), en ze altijd 24/7 beschikbaar zijn.

Dit betekent dus ook dat vaak alles geregeld wordt door Amazon en niet door een developer zelf, dit maakt het makkelijk om een tool te maken die kan omgaan met Big Data zonder dat er veel geld of tijd erin gestopt hoeft te worden. Voor sommige tools hoef je niet eens iets te coderen maar kan het al gewoon werken via de AWS Dashboard.

Hoe helpt AWS met het omgaan van Big Data?

Voor het omgaan met Big Data hebben wij eerst een plek nodig om deze data op te slaan, hiervoor kunnen we de tool AWS Data Lake gebruiken. Deze Data Lake kan omgaan met beide statische en real-time data en verbindt gelijk met verschillende Big Data gefocuste tools.

Een voorbeeld van hoe de Data Lake met andere tools en data verbindt.

Een van deze Big Data gefocuste tools dat ook erg belangrijk is, vooral sinds je de GDPR-regels moet volgen in Europa, is de DIDL (De-Identified Data Lake). Deze zorgt ervoor dat jij jouw ingeladen data al gelijk kunt omvormen om GDPR-regels te volgen wanneer je met jouw data omgaat.

Wat je gelijk daarna kunt gebruiken is hun toolset genaamd Amazon Kinesis, deze tools maken het makkelijker om inzicht te krijgen en een analyze te maken op jouw data.

Een voorbeeld van hoe AWS Kinesis kan gebruikt worden in een winkelsysteem.

Amazon biedt hier ook meerdere verschillende ingebouwde machine learning tools die het makkelijker maakt om deze te implementeren.

Bijvoorbeeld, zij hebben een tool die het mogelijk maakt om automatisch met machine-learning erachter te komen wat er op een bepaalde foto staat en deze data gelijk weer in de data storage op te slaan, dit kan gedaan worden met maar 1 to 3 extra lijnen code.

Dit is alles dat nodig is om een foto te herkennen en labels terug te geven.

Normaal zou het maken van een neuraal netwerk dat automatisch kan uitvinden wat er op een foto staat dagen/weken lang duren, maar AWS geeft ons de mogelijkheid om dit op te zetten in een zeer korte tijd.

Dit is maar een van de velen tools die AWS aanbiedt voor het versimpelen van het opslaan, opschonen en omvormen van de data tot iets dat iedereen kan gebruiken.

De gastles, gegeven door Dick Eimers, heeft een zeer goed voorbeeld gegeven van een paar van deze tools en liet zien dat het maken van een Big Data applicatie minder tijd kost en makkelijker is vanwege deze opties die Amazon ons aanbieden.
